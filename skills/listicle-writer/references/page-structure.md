# Page Structure

## Product Introduction

Appears after the final listicle point. Introduces the product for readers who scrolled through the arguments and want to know more.

**Include:**
- What is it
- What's in it (ingredients, components, materials)
- Who makes it
- Key differentiators (what makes it different from other products in this category)

This section is a natural home for adjacent benefits, features that are relevant to the persona but weren't core enough to the theme to become listicle points. Weave them in where they add value.

**Word count:** 50-100 words.

**Headline options (select based on zone and mode):**
- "What Is [Product]?" (neutral, any zone)
- "Meet [Product]" (Zone 1-2)
- "About [Product]" (neutral, any zone)

## FAQ Section

3-6 common questions. Prioritize purchase-friction questions over educational ones.

**Source questions from:**
- Gate 1 research: customer reviews, forum discussions, common objections identified during competitive analysis
- Purchase hesitations: "Can I use it with X?", "How long until results?", "What if it doesn't work?"
- Shipping/logistics: "How long does delivery take?", "What's the return policy?"
- Adjacent benefits that didn't make it into the listicle points can be surfaced as FAQ answers where relevant (e.g., "Does it help with back pain?" if back pain reduction is an adjacent benefit)

**Format:**

```
## Frequently Asked Questions

**[Question 1]**
[Answer, 1-3 sentences]

**[Question 2]**
[Answer, 1-3 sentences]

**[Question 3]**
[Answer, 1-3 sentences]
```

Short answers, 1-3 sentences each. Do not repeat sales arguments from the listicle points. The FAQ handles objections, the listicle handles persuasion.

## CTA Section

**Include:**
- Product image placeholder
- Price and key trust signals (warranty, guarantee)
- 3-5 checkmark features
- CTA button

**CTA section word count:** 30-50 words.

**Format:**

```
## [Action-Oriented Headline]

[ IMG HERE ]

[Price]. [Warranty/guarantee].

✓ [Feature 1]
✓ [Feature 2]
✓ [Feature 3]
✓ [Feature 4]
✓ [Feature 5]

**[CTA BUTTON: Action Text]**
```

## Comparison Table Rules

Use comparison tables for competitive arguments where a side-by-side visual makes the argument land faster than prose.

**Rules:**
- Max 4-5 columns, max 3-4 rows
- Every cell should be one word or Yes/No
- If a cell needs a sentence, it's not a table. Write it as prose instead.
- Use tables only when the competitive argument genuinely benefits from visual side-by-side
- Competitors in the table should be the ones relevant to the specific feature being compared, not a generic list of all competitors

**Example:**

```
| | [Product] | Floor Planks | Ab Roller | Sit-Ups |
|---|---|---|---|---|
| Core activation | Full | Partial | Partial | Minimal |
| Wrist strain | None | High | High | None |
| Setup time | 10 sec | None | 30 sec | None |
```

## Formatting Rules

1. **One sentence per line within points.** No stacking sentences into paragraphs. Each thought gets its own line for mobile scannability.
2. **Image placeholder before every point headline.** Format: [ IMG HERE ]
3. **Generous whitespace between points.** Use section dividers (---) between points.
4. **Bold point headlines.** Format: **#[X]: [Headline]**
5. **Generic bullets in Product Introduction.** Use - not emojis.
6. **Checkmarks in CTA section only.** Use ✓ for feature lists.
7. **No em-dashes anywhere.** Use periods or commas.
8. **No dense text blocks anywhere.** If any section looks like a paragraph on mobile, add line breaks.
9. **Comparison tables where competitive arguments benefit from visual side-by-side.** Follow the table rules above.
